# PBL-Gaokao System

**A Project-Based Learning framework for ages 2-18, fully aligned with China's 高考 system, built for the AI era.**

## The Problem

China's education system produces students who can pass exams but often lose their intrinsic motivation, curiosity, and sense of agency along the way. Meanwhile, the AI era demands exactly those qualities — the ability to frame problems, think critically, and say "I can figure this out."

The standard response is to abandon the exam system for "alternative education." This is a false choice. The knowledge tested in 高考 is not the problem. **Decontextualized drilling** is the problem.

## The Thesis

> Every 高考 knowledge point can be re-encountered as a real problem worth solving.
> The curriculum becomes the tool. The problem becomes the entry point.
> The student's inner narrative shifts from "我必须背这个" to **"我可以用这个解决..."**

This project designs that system — from age 2 through age 18.

## Core Principles

1. **Full 高考 Coverage** — No knowledge point is skipped. Students following this system are exam-competitive.
2. **Problem-First Learning** — Every topic is introduced through a genuine problem, not an abstract lesson.
3. **主体性 (Agency)** — "我可以..." is the measurable outcome at every stage. Students own their learning.
4. **Motivation by Design** — Intrinsic drive is architecturally sustained, not left to chance.
5. **AI as Collaborator** — Students learn to work with AI tools as amplifiers, not replacements.
6. **Honest About Constraints** — The 高考 timeline is real. This system makes exam prep more effective, not optional.

## Age Stages

| Stage | Ages | Name | Core Mode |
|-------|------|------|-----------|
| 1 | 2-5 | Seedling | Sensory exploration, play-based inquiry, language immersion |
| 2 | 6-9 | Explorer | Structured curiosity, first projects, foundational skills through problems |
| 3 | 10-12 | Builder | Multi-week projects, cross-subject integration, growing independence |
| 4 | 13-15 | Investigator | Research-driven projects, 中考 alignment, identity formation |
| 5 | 16-18 | Architect | Capstone projects, 高考 synthesis, AI-augmented deep work |

## The Key Mechanism: Test → Problem Transformation

Instead of:
> "Today we learn quadratic equations. Open your textbook to page 47."

We have:
> "A local farmer wants to maximize the area of a rectangular pen using 40 meters of fencing against a wall. What dimensions should she use? Let's figure it out."

The student discovers quadratic optimization because they need it, not because it's on the syllabus. But it IS on the syllabus — and now they understand it deeply.

This transformation is applied **systematically** to every required knowledge point across all subjects.
